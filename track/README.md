# 项目追踪

这个目录是 Learnpathly 的轻量“项目运维/进度日志”。建议每个主题都持续维护：
- 当前状态：哪些在运行、哪些卡住了
- 关键决策：为什么这样做
- 风险：可能会在哪里出问题
- 下一步：明确可执行的行动项

## 追踪主题
- [前端](./frontend.md)
- [后端](./backend.md)
- [数据库](./database.md)
- [AI 功能](./ai.md)
- [部署](./deployment.md)

## 建议扩展（可新增更多追踪文件）
- `product.md`：目标、用户画像、关键流程、指标
- `admin.md`：管理端工作流、权限、内容运营
- `observability.md`：日志/指标/链路追踪、告警、SLO
- `security.md`：鉴权、RBAC、密钥管理、风险记录
- `performance.md`：性能热点、预算、压测与 profiling
- `qa.md`：测试计划、回归清单、已知不稳定点
- `content.md`：主题生产流水线、审核/发布规则、内容规范
- `roadmap.md`：里程碑、阶段拆解、边界与取舍
